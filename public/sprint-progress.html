Product & Sprint Review — March 11, 2026

What Layers Does

The user journey from signup to daily use

1
Sign up & onboard
Email/password or Google OAuth at /signup. Guided onboarding wizard walks you through org creation, first integration, and first chat.
↓
2
Bring your knowledge in
Upload PDFs, DOCX, TXT, Markdown directly. Or Connect Google Drive, GitHub, Slack, Linear via OAuth. Each document is AI-extracted and vector-embedded via a multi-stage context pipeline.
↓
3
Chat with your knowledge
Agentic AI chat at /chat. 7 models (Claude, GPT-4o, Gemini). Hybrid search retrieves relevant docs. Multi-step tool calling with citations linking back to sources.
↓
4
Organize into Sessions
Create named workspaces at /sessions. Link specific documents — chat is scoped to those docs only. Great for projects and research threads.
↓
5
Stay on top of action items
Inbox at /inbox — AI surfaces action items, decisions, and mentions from every document. Daily cron generates prioritized items (urgent/high/normal/low).
↓
6
Monitor quality
Analytics at /analytics — Context Health, Agent Analytics (search quality, errors, tokens), Retrieval Quality scoring.

How It Works

Visual walkthroughs of key features and data flows

Context Ingestion Pipeline Core
When you upload a file or sync an integration, it flows through a multi-stage processing pipeline:
Upload / Sync
→
AI Extraction
→
Title + Summary
→
Vector Embedding
→
Searchable Context
// Upload triggers the pipeline automatically
POST /api/context/upload
// Body: FormData with file (PDF, DOCX, TXT, MD)
// Response: { id, title, summary, status: "processing" }

// Or sync from integrations
POST /api/integrations/google-drive/sync
// Fetches docs, exports as text, runs same pipeline
Agentic Chat Flow AI-Powered
The chat agent uses a tool-calling loop — it searches your context, retrieves documents, and synthesizes answers with citations:
User Message
→
LLM Reasoning
→
Tool Call:
search_context
→
Hybrid Search
(Vector + BM25)
→
Ranked Results
+ Citations
→
Final Answer
// Example: User asks about a project deadline
"When is the Layers launch date?"

// Agent thinks -> calls search_context("Layers launch date")
// Hybrid search: vector similarity + BM25 keyword matching
// Reciprocal Rank Fusion combines scores
// Agent reads top results -> synthesizes answer with [1][2] citations
Session-Scoped Workspaces Collaboration
Sessions narrow the AI's focus to specific documents — like a project room:
Create Session
"Q1 Planning"
→
Link 5 docs
(drag & drop)
→
Chat in session
→
Search scoped
to linked docs only
// Create a focused workspace
POST /api/sessions { "name": "Q1 Planning", "goal": "Finalize roadmap" }

// Link specific context items
POST /api/sessions/:id/context { "context_ids": ["abc", "def"] }

// Chat only searches those docs — no noise from other projects
