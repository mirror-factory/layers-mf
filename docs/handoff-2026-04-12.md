## What Was Built

### Portal Viewer (`/portal/[token]`)
An AI-powered document collaboration portal where clients receive proposals and can talk to them, understand them, visualize them, and interact hands-free via voice.

**URL:** `http://localhost:3000/portal/bluewave-demo`

### Core Architecture

| Component | Technology | Purpose |
|-----------|-----------|---------|
| Document Rendering | PDF.js via react-pdf | All docs (DOCX, XLSX, PDF) converted to PDF at build time via LibreOffice |
| Chat | ChatInterface with ChatVariant config | Reusable, typed config system for styling/behavior |
| Voice | Browser STT + Cartesia sonic-turbo TTS | 40ms first-byte latency, barge-in support |
| AI Model | Gemini 3.1 Flash Lite (testing) | Via Vercel AI Gateway |
| Tools | 20+ AI tools | Highlight, navigate, chart, walkthrough, bookmark, compare, brief, etc. |
| Splash | PortalSplash component | Configurable logo, branding, subtitle, colors |

### Everything is PDF
All DOCX and XLSX files are pre-converted to PDF using LibreOffice headless (`scripts/convert-docs-to-pdf.ts`). This means:
- One renderer for all doc types (PortalPdfViewer)
- Highlight, scroll, annotate, two-column spread work identically on everything
- Text layer search is consistent
- No more docx-preview/jspreadsheet dependencies for viewing
- Original files kept for download links

### ChatVariant System
The chat component accepts a `variant` prop with typed config:
```tsx
interface ChatVariant {
  style: "default" | "portal";
  headingColor, bodyColor, mutedColor: string;
  inputBorder, inputBg: string;
  gradientFrom, gradientTo: string;
  suggestions: { text: string; accent: boolean }[];
  voiceEnabled: boolean;
  tools: string[];
}
```
Exported: `PORTAL_VARIANT`, `DEFAULT_VARIANT`, `ChatVariant` type.

### Intent Detection System
Server-side keyword matching narrows available tools per message to prevent flash-lite from looping. History-referenced tools always kept for deserialization:

```
"highlight budget" → only highlight_text + switch_document (2 tools)
"chart the phases" → only render_chart (1 tool)
"walk me through"  → only walkthrough_document (1 tool)
```

### Voice Mode
- **STT:** Browser native `webkitSpeechRecognition` (free, instant)
- **TTS:** Cartesia sonic-turbo via `/api/tts` route (40ms first-byte)
- **Voice ID:** Jillian - Happy Spirit (cheerful, upbeat)
- **Barge-in:** Recognition restarts during TTS, stops audio on speech
- **UI:** Compact waveform bars flanking mic button, inline in chat input area

## Architecture Decisions

### Why PDF for everything?
Three separate renderers (PDF.js, docx-preview, jspreadsheet) had different DOM structures, making highlight/scroll/annotate work differently on each. Converting everything to PDF unified the experience.

### Why intent detection instead of smarter models?
Gemini 3.1 Flash Lite is cheap but weak at tool selection. With 20+ tools available, it loops through search/lookup tools. Intent detection narrows to 1-3 tools per request, forcing correct selection. Upgrading to gemini-3.0-flash would reduce the need for this.

### Why ChatVariant instead of separate components?
The chat is used in portal mode, will be used in the AI starter kit, and potentially in other contexts. A typed variant config allows customization without forking the component.

### Why Cartesia over ElevenLabs?
Cartesia sonic-turbo has 40ms first-byte latency (vs 75ms for ElevenLabs Flash v2.5). Both are in the Vercel ecosystem. ElevenLabs has native AI SDK support; Cartesia required a custom `/api/tts` route.
